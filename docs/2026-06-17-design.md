# inspiration-scout — design spec

Data: 2026-06-17
Status: zatwierdzony (design), gotowy do planu implementacji

## Cel

Skill dla Claude Code, który po podaniu branży przeszukuje galerie inspiracji
designerskich po angielskich hasłach związanych z branżą i zapisuje materiał
referencyjny: pełnostronicowe zrzuty żywych stron (z podstronami), grafiki
mockupów, albo tylko wybrany typ sekcji / jedną podstronę.

## Zakres galerii

Dwa typy, traktowane różnie (hybryda):

- **Galerie żywych stron** — `godly.website`, `awwwards.com`, `cosmos.so`,
  `framer.com/marketplace` (szablony Framera), `webflow.com/templates` (szablony
  Webflow): prowadzą do realnych witryn / żywych preview szablonów → full-page
  screenshoty + crawl podstron. Szablony Framera/Webflow są zwykle multipage —
  podstrony i targetowanie sekcji działają tu szczególnie dobrze.
- **Galerie mockupów** — `dribbble.com`, `behance.net`, `pinterest.com`:
  „inspiracja" jest już grafiką → pobieranie grafiki w najwyższej dostępnej
  rozdzielczości, bez podstron i bez targetowania sekcji. Pinterest dodatkowo
  może odnotować link do źródła pina w manifeście.

## Decyzje (ustalone z użytkownikiem)

1. **Hybryda** — żywe strony screenshotujemy, mockupy pobieramy jako grafiki.
2. **Sesja zalogowana** — trwały profil przeglądarki; jednorazowy ręczny login
   odblokowuje treść gated (cosmos / dribbble / behance).
3. **Output: folder + contact sheet** — PNG/grafiki + `index.html` (kontaktówka
   miniatur z linkami do źródeł) + `manifest.json`.
4. **Limity zachowawcze** — domyślnie ~5 stron/galeria, podstrony głębokość 1;
   przestrajalne per uruchomienie.
5. **Targetowanie sekcji** — najpierw wnioskowanie z treści (skan DOM); gdy się
   nie da — fallback na typową podstronę, a w ostateczności full-page + log.

## Architektura

Samodzielny skrypt Python + Playwright sterowany przez `SKILL.md`. Generowanie
angielskich słów kluczowych z branży należy do Claude'a (podawane skryptowi
jako argument), nie do sztywnej mapy w kodzie.

```
.agents/skills/inspiration-scout/
  SKILL.md          # orkiestracja: Claude generuje keywords, dobiera tryb, woła scout.py
  scout.py          # CLI orchestrator (argumenty, pętla galeria → capture → report)
  galleries.py      # adaptery per galeria: budowa URL wyszukiwania + ekstrakcja wyników
  capture.py        # Playwright: full-page, crawl podstron (depth 1), targetowanie sekcji/podstrony
  download.py       # pobieranie grafik mockupów (dribbble/behance)
  report.py         # render index.html (kontaktówka) + zapis manifest.json
  .browser-profile/ # trwały profil dla sesji zalogowanej (gitignored)
```

Symlink w `.claude/skills/inspiration-scout -> ../../.agents/skills/inspiration-scout`
(zgodnie z istniejącą konwencją workspace).

### Jednostki i odpowiedzialności

- **galleries.py** — dla każdej galerii: `search_url(keyword)` oraz
  `extract_items(page) -> [Item]`, gdzie `Item = {gallery, kind: "site"|"mockup",
  url, title}`. Granica: zna wyłącznie strukturę HTML/URL danej galerii.
- **capture.py** — `capture_site(context, item, mode, depth, section/subpage) ->
  [Asset]`. Pełna strona, crawl podstron depth 1, targetowanie sekcji. Nie wie
  nic o galeriach poza polami `Item`.
- **download.py** — `download_mockup(context, item) -> [Asset]`. Tylko grafiki.
- **report.py** — `write_report(out_dir, assets, run_meta)` → `index.html` +
  `manifest.json`. Czyste wejście/wyjście, łatwe do unit-testu.
- **scout.py** — parsowanie argumentów, dobór galerii, pętla, agregacja, wołanie
  report. Komenda `login <gallery>` (headed) do zapisania sesji.

## Przepływ danych

1. Użytkownik podaje branżę + tryb (+ opcjonalnie podzbiór galerii).
2. Claude generuje listę angielskich haseł (synonimy/warianty) i woła `scout.py`
   z `--industry` i `--keywords`.
3. **Discovery** — dla każdej włączonej galerii: otwarcie URL wyszukiwania,
   ekstrakcja kandydatów (`site` lub `mockup`) z limitem (~5).
4. **Capture**:
   - żywe strony → full-page PNG; w trybie `full` crawl podstron depth 1 (cap);
     w trybie `section`/`subpage` capture celowany.
   - mockupy → pobranie grafiki w najwyższej rozdzielczości.
5. **Report** — zapis do `./inspiration-output/<branza>-<data>/`: assety +
   `index.html` + `manifest.json`.

## Tryby (CLI)

```
scout.py --industry "dental clinic" --keywords "dentist,dental clinic,orthodontist" \
         --galleries godly,awwwards,cosmos,framer,webflow,dribbble,behance,pinterest \
         --mode full --limit 5 --depth 1
scout.py ... --mode section --section testimonials
scout.py ... --mode subpage --subpage /services
scout.py login cosmos
```

Tryby: `full` (cała strona + podstrony), `section` (wybrany typ sekcji),
`subpage` (jedna podstrona). `section`/`subpage` dotyczą tylko żywych stron;
dla mockupów ignorowane (zawsze pobranie grafiki).

### Logika targetowania sekcji (`--mode section`)

1. **Wnioskowanie z treści** — skan DOM po nagłówkach/landmarkach/anchorach i
   słowach kluczowych typu sekcji (`testimonials`→{testimonials, reviews,
   opinie...}, `services`→{services, offer...}, `form`→{form, contact, kontakt},
   `pricing`→{pricing, plans}). Zrzut dopasowanego elementu (element screenshot).
2. **Fallback podstrona** — gdy nie da się wywnioskować w obrębie strony: próba
   typowej ścieżki (`/services`, `/contact`, `/pricing`...) i zrzut tej podstrony.
3. **Ostateczność** — full-page screenshot + wpis w manifeście „section not found".

## Sesja zalogowana

Playwright `launch_persistent_context(user_data_dir=.browser-profile)`. Komenda
`scout.py login <gallery>` otwiera headed browser; użytkownik loguje się ręcznie
raz, ciasteczka zostają w profilu. Kolejne uruchomienia (headless) korzystają z
sesji. `.browser-profile/` w `.gitignore`.

## Obsługa błędów i grzeczność

- Każdy element w try/except — `blocked` / `login-required` / `timeout` /
  `section-not-found` trafia do `manifest.json` ze statusem; reszta leci dalej.
- Realistyczny User-Agent, drobny delay między żądaniami, współdzielony kontekst
  przeglądarki, rozsądne timeouty nawigacji.
- Brak prób obchodzenia zabezpieczeń ponad zwykłą sesję zalogowaną; treść
  niedostępna jest logowana jako pominięta, nie forsowana.

## Output

```
inspiration-output/<branza>-<YYYY-MM-DD>/
  <gallery>__<slug>__full.png
  <gallery>__<slug>__<subpage>.png
  <gallery>__<slug>.jpg            # mockup
  index.html                       # kontaktówka miniatur + linki do źródeł, galeria, hasło
  manifest.json                    # [{gallery, source_url, keyword, mode, files[], status}]
```

`manifest.json` jest źródłem prawdy; `index.html` renderowany z manifestu.

## Testy

- **Unit (z fixturkami HTML)**: `search_url` per galeria, `extract_items` na
  zapisanym HTML, budowa `manifest.json`, render `index.html`.
- **Smoke**: jedno realne uruchomienie na `godly.website` (najodporniejsza na
  anti-bot).
- Pozostałe galerie bywają kruche (anti-bot, zmiany DOM, login) — to ograniczenie
  jest jawnie odnotowane; skill raportuje pominięcia zamiast udawać niezawodność.

## Poza zakresem (YAGNI)

- Analiza/ocena designu pobranych stron (to robi `extract-design`).
- Omijanie CAPTCHA/anti-botów ponad sesję zalogowaną.
- Równoległość wielowątkowa — sekwencyjnie, grzecznie wobec serwisów.
- Harmonogramowanie/automatyczne odświeżanie.
